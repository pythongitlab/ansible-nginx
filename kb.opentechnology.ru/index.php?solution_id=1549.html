    Сергей Гусев
    18.09.2025
    0 Комментариев
    Интеграция позволяет осуществить печать документов установленного образца на бланках строгой отчетности, а также подготовить файл с данными этих документов для выгрузки в федеральный реестр сведений о документах об образовании (о квалификации, об обучении).
    Функционал, описанный в данной статье, в полном объеме доступен в СЭО 3КL, начиная с версии 4.1.5с.
    Если вы используете более раннюю версию системы — обратитесь с заявкой на проведение обновления в службу технической поддержки ООО «Открытые технологии».

    1. Общая информация

    В рамках интеграции с федеральной информационной системой «Федеральный реестр сведений о документах об образовании и (или) о квалификации, документах об обучении» (ФИС ФРДО) в СЭО 3КL реализован набор настроек и сценариев, которые предназначены для:

    • генерации и печати документов на бланках строгой отчетности;
    • формирования файла с данными этих документов для последующей загрузки в федеральную информационную систему.

    -Вернуться к содержанию-

    1.1. Общие условия интеграции

    Для корректной отработки сценария необходимо, чтобы предварительно были выполнены следующие действия:

    • образовательная организация подключена к ФИС ФРДО;
    • в организации назначен ответственный специалист, отвечающий за работу с бланками строгой отчетности и передачу данных в ФИС ФРДО, в СЭО 3КL для него создана отдельная роль, наделенная всеми необходимыми правами;
    • в системе созданы и верно заполнены все необходимые поля профиля пользователя;
    • в системе выполнена глобальная настройка плагина «Сертификат СЭО 3КL»;
    • в курсе выполнена настройка сертификата и условий его выдачи, активирована опция «Печать на бланке включена»;
    • в Панели управления СЭО 3КL присутствует предустановленная вкладка «Выгрузка для ФИС ФРДО»;
    • шаблон файла для выгрузки сертификатов в ФИС ФРДО соответствует последним требованиям федеральной информационной системы.

    -Вернуться к содержанию-

    1.2. Права

    Право печати сертификатов на бланке (mod/simplecertificate:letterheadprint). По умолчанию присутствует у диспетчера-администратора.

    При отсутствии у пользователя этого права столбец печати на бланке в таблице выданных сертификатов в интерфейсе сертификата не будет отображаться, а при переходе к интерфейсу печати по прямой ссылке пользователь будет перенаправлен с сообщением о невозможности выполнить действие.

    Право для изменения данных сертификата после выгрузки в ФИС ФРДО (mod/simplecertificate:modifyafterfrdoloading). По умолчанию не доступно ни одной из ролей.

    Право дает возможность:

    • переопределять данные выгруженного сертификата, в том числе данные печати;
    • удалять выгруженные сертификаты.
    Администраторы СЭО 3КL в любом случае будут иметь возможность переопределять данные выгруженного сертификата и удалять выгруженные сертификаты даже при отсутствии права для изменения данных сертификата после выгрузки в ФИС ФРДО благодаря наличию у них суперправа moodle / site: doanything.

    «Определение ролей, настройка прав»
    «Роли, контекст, ролевая модель»

    2. Настройка интеграции с ФИС ФРДО

    Задача:

    В СЭО 3КL центра ДПО необходимо настроить выдачу удостоверений о повышении квалификации на бланках строгой отчетности (Рис. 2.1) и формирование файла с информацией о выданных удостоверениях для федеральной информационной системы «ФИС ФРДО».

    Рис. 2.1. Удостоверение о повышении квалификации, выдаваемое центром ДПО. Пример.
    Подразумевается, что центр ДПО на момент реализации решения уже подключен к ФИС ФРДО.

    Решение:

    1) В организации назначается (при его отсутствии) ответственный специалист, отвечающий за работу с бланками строгой отчетности и передачу данных в ФИС ФРДО. В СЭО 3КL для него создается отдельная роль, наделенная всеми необходимыми правами.

    Для создания такой роли необходимо обратиться с соответствующей заявкой в техническую поддержку ООО «Открытые технологии». При этом набор необходимых для роли прав (уровень «вмешательства» ответственного специалиста в СЭО 3КL) будет определяться, в первую очередь, внутренними регламентами вашей организации и все последующие действия, описанные в решении, ответственный специалист выполняет самостоятельно, либо при помощи диспетчера-администратора системы.

    2) Ответственный специалист проверяет (Администрирование->Пользователи->Учетные записи->Дополнительные поля профиля) наличие в системе необходимых для корректной генерации файла выгрузки в ФИС ФРДО дополнительных полей профиля пользователя (Рис. 2.2) и, в случае их отсутствия, создает и заполняет их:

    Название поля Описание Примечание
    baseeducationlevel Уровень образования (ВО/СПО) Выпадающий список
    basedocumentlastname Фамилия, указываемая в дипломе о ВО/СПО (если оставить незаполненным, в документ будет подставляться фамилия из основного поля) Текстовое поле, до 100 символов
    basedocumentserial Серия документа о ВО/СПО Текстовое поле, до 16 символов
    basedocumentnumber Номер документа о ВО/СПО Текстовое поле, до 16 символов
    birthday Дата рождения получателя документа Дата формата дд.мм.гггг
    sex Пол получателя Выпадающий список
    snils СНИЛС Формат ввода: 111-222-333 00
    citizenship Гражданство получателя Выпадающий список, коды стран по ОКСМ

    Рис. 2.2. Дополнительные поля профиля, необходимые для корректной генерации файла выгрузки в ФИС ФРДО.
    Обратите внимание: если к моменту формирования файла для ФИС ФРДО эти поля будут отсутствовать в системе или не будут заполнены у пользователей (Рис. 2.3), чьи данные выгружаются в федеральную информационную систему, то соответствующие столбцы в файле выгрузки останутся пустыми и таблицу придется дозаполнять вручную.

    Рис. 2.3. Пример профиля пользователя с заполненными необходимыми полями.
    Приведенный перечень полей актуален для шаблона по умолчанию. Вы можете использовать в вашей системе поля с иными названиями, но в этом случае для корректного создания файла выгрузки необходимо изменить параметры шаблона.
    Подробная информация по созданию дополнительных полей профиля в статье «Добавление кастомных (настраиваемых) полей в профиль пользователя».

    3) Ответственный специалист определяет глобальные настройки плагина «Сертификат СЭО 3КL» (Администрирование->Плагины->Модули элементов курса->Сертификат СЭО 3КL):

    Выводить QR-код сертификата — опция не установлена

    Печать на бланке включена — опция установлена
    Вид документа = Удостоверение о повышении квалификации
    Дополнительная профессиональная программа = Повышение квалификации

    Год начала обучения для документа о квалификации = 2022

    4) Ответственный специалист проверяет актуальность шаблона файла для выгрузки в ФИС ФРДО и при необходимости переопределяет его.

    Если у вас дополнительные поля с данными для выгрузки в ФИС ФРДО настроены так же, как в предлагаемом решении — переопределять шаблон файла для выгрузки не нужно.
    Если же в вашей системе эти дополнительные поля называются иначе - необходимо изменить шаблон файла для выгрузки по следующему алгоритму:
    1) Скопировать в подходящий текстовый редактор текст шаблона.
    2) Внести в текст соответствующие правки (заменить названия полей по умолчанию на актуальные).
    3) Вставить отредактированный текст шаблона в поле «Переопределить шаблон файла для выгрузки сертификатов в ФИС ФРДО» глобальных настроек плагина и сохранить изменения.

    5) Ответственный специалист создает в курсе элемент курса «Сертификат СЭО 3КL», настраивает его:

    Название сертификата = Удостоверение о повышении квалификации
    Файл с изображением сертификата = Файл подложки
    Текст сертификата = Текстовый файл с html-кодом сертификата (скопированный из файла код необходимо вставить в окно редактора в режиме работы с html-кодом)
    Включить обратную сторону сертификата  = Нет
    Альтернативное название курса = Оказание первой помощи
    Альтернативный серийный номер сертификата = Номер с серией в формате А001-ХХХХ-ХХХХ
    Серия для кода = А001
    Пытаться автоматически сгенерировать при изменении оценки за курс = Да
    Доставка = Не отправлять студенту сертификат, пользователь получит сертификат иным способом
    Печать на бланке включена — опция установлена

    6) Ответственный специалист, используя ксерокопии номерного бланка и оборудование (принтер), на котором впоследствии будет выполняться печать , проверяет точность наложения текста документа на бланк при печати. При необходимости вносит правки в настройки документа таким образом, чтобы все поля текста документа точно попадали в нужные места удостоверения (Рис. 2.4).

    Рис. 2.4. Пример пробного сертификата. распечатанного на ксерокопии бланка.

    7) Ответственный специалист проверяет наличие в Панели управления СЭО 3КL вкладки «Выгрузка для ФИС ФРДО» (Рис. 2.5) и при необходимости создает ее.

    Рис. 2.5. Пример интерфейса Панели управления СЭО 3КL.

    Результат:

    1) Пользователи успешно завершают прохождение курса повышения квалификации и система генерирует им сертификаты (Рис. 2.6).

    Рис. 2.6. Страница личного кабинета пользователя с выданным сертификатом.

    2) Ответственный специалист настраивает на вкладке «Выгрузка для ФИС ФРДО» отбор сертификатов, которые были выданы в СЭО 3КL за текущий отчетный период, но еще не распечатывались (Рис. 2.7):

    Подлежит печати = Да
    Дата печати заполнена = Нет
    Данные уже выгружались в ФИС ФРДО = Нет

    Подробнее об инструменте фильтрации в разделе «Использование фильтра на вкладке «Выгрузка для ФИС ФРДО».

    Рис. 2.7. Вкладка «Выгрузка в ФИС ФРДО» с отфильтрованными сертификатами, которые подлежат печати, но еще не распечатывались и данные о которых не передавались в ФИС ФРДО.

    3) Ответственный специалист по ссылке «Распечатать сертификат» (см. поз. 1 Рис. 2.7) переходит на страницу «Печать сертификата на бланке» (Рис. 2.8). Здесь при необходимости корректирует данные документа в соответствии с приказом по учебному заведению (поз. 1. Рис. 2.8) и нажатием на кнопку «Сохранить» записывает измененные данные.

    Рис. 2.8. Страница «Печать сертификата на бланке».

    4) Ответственный специалист, нажав на ссылку «Скачать файл сертификата без бланка» (см. поз. 2 Рис. 2.8), переходит на вкладку со сгенерированным файлом для печати на бланке строгой отчетности и еще раз проверяет правильность заполнения документа (Рис. 2.9).

    Обратите внимание: файл в этом случае генерируется без подложки, так как будет распечатываться на номерном бланке.

    5) Ответственный специалист распечатывает проверенный файл документа на номерном бланке (Рис. 2.10).

    Рис. 2.10. Пример распечатанного свидетельства на бланке с серией АААА и номером 12345.

    6) Ответственный специалист возвращается на страницу «Печать сертификата на бланке» (см. Рис. 2.8) и вносит информацию о серии и номере распечатанного документа в поля «Серия бланка» и «Номер бланка» (Рис. 2.11). Нажав «Сохранить и вернуться» записывает эти данные в СЭО 3КL.

    Рис. 2.11. Внесение данных о номере и серии бланка распечатанного документа.
    Описываемая в пунктах 3) - 6) последовательность действий не является безусловной и будет зависеть от того, как выстроен процесс подготовки документов в вашей организации.

    7) Ответственный специалист повторяет действия, описанные в пунктах 3)-6) для всех пользователей, которым необходимо распечатать сертификаты.

    8) Ответственный специалист настраивает на вкладке «Выгрузка для ФИС ФРДО» отбор сертификатов, которые были выданы в СЭО 3КL за текущий отчетный период, уже распечатаны, но еще не переданы в ФИС ФРДО:

    Подлежит печати = Да
    Дата печати заполнена = Да
    Дата начала периода и Дата окончания периода = границы отчетного периода (01.11.2022г. — 02.12.2022г. в примере)
    Данные уже выгружались в ФИС ФРДО = Нет

    9) Ответственный специалист, выделяет (отмечает галочками) все сертификаты, данные о которых необходимо передать в ФИС ФРДО, и выполняет массовое действие «Сформировать файл для выгрузки в ФИС ФРДО» (Рис. 2.12).

    Рис. 2.12. Массовое действие «Сформировать файл для выгрузки в ФИС ФРДО» для выбранных сертификатов на вкладке «Выгрузка для ФИС ФРДО».

    8) Ответственный специалист во вновь открывшемся окне нажимает кнопку «Сформировать файл» (Рис. 2.13) и запускает процесс генерации файла выгрузки, которых будет сохранен на его компьютере (место сохранения зависит от браузера и его настроек, в большинстве случаев, это папка «Загрузки»).

    3.2. Создание вкладки «Выгрузка для ФИС ФРДО» Панели управления СЭО 3КL

    Инструмент «Панель управления СЭО 3КL» доступен в Настройки->Администрирование->Панель управления СЭО 3КL. Вкладка «Выгрузка для ФИС ФРДО» добавлена в Панель управления СЭО 3КL вкладкой по умолчанию, начиная с релиза 3.9.18а.

    Если вы пользуетесь более ранней версией СЭО 3КL — обратитесь в техническую поддержку ООО «Открытые технологии» с заявкой на обновление.

    Если ваша СЭО 3КL версии 3.9.18а (либо выше) и по каким-то причинам в Панели управления СЭО 3КL этой вкладки нет, ее можно сформировать следующими способами:

    Первый способ:
    Выполнить сброс вкладок к предустановленному состоянию (перейти на страницу «Конфигурация» и нажать кнопку «Вернуть настройки по умолчанию»).

    Если у вас ранее были созданы какие-либо кастомные вкладки или модифицированы предустановленные, после сброса они удалятся! Чтобы не потерять их, перед сбросом  рекомендуется скопировать yaml-код из поля «Конфигурация панели управления СЭО 3KL» (О пользователе->Редактирование настраиваемых полей).

    Второй способ:
    Добавить yaml-код вкладки в поле «Конфигурация панели управления СЭО 3KL» (О пользователе->Подробная информация о пользователе->Редактирование настраиваемых полей).

    Обратите внимание: чтобы не удалить имеющиеся у вас кастомные вкладки, yaml-код необходимо именно добавлять к уже существующему в поле «Конфигурация панели управления СЭО 3KL» коду, а не замещать его.
    Если поле «Конфигурация панели управления СЭО 3KL» пустое, это означает, что пользователем еще не выполнялось никаких действий по изменению/добавлению вкладок. В этом случае необходимо отредактировать через конфигуратор любую из существующих вкладок (например, добавить в нее новое поле) или создать новую пользовательскую вкладку. После этого yaml-код всех вкладок отобразится в поле «Конфигурация панели управления СЭО 3KL».

    3.5. Отмена печати

    Для того, чтобы отменить печать, необходимо на странице «Печать сертификата на бланке» очистить поля «Серия бланка» и «Номер бланка», установить опцию «Подтвердить сохранение новых данных печати» и нажать кнопку «Сохранить».

    После этого дата печати будет очищена, сообщение в столбце «Печать на бланке» вкладки «Выгрузка для ФИС ФРДО» Панели управления СЭО 3KL станет иметь вид: Сертификат еще не печатался на бланке.

    При изменении или удалении серии и номера бланка сохраняется слепок предыдущих данных. Таким образом в системе хранится информация о всех бланках, которые были использованы для печати этого сертификата.

    3.6. Выгрузка в ФИС ФРДО удаленных из системы сертификатов

    В ФИС ФРДО можно выгрузить удаленные сертификаты и сертификаты удаленных модулей (certissue, помеченные удаленными) при условии, что они были распечатаны (был сформирован файл) до удаления.

    Распечатать удаленный сертификат нельзя, так как для формирования файла нужны данные из настроек модуля.

    4.1. Создание в СЭО 3KL удостоверения о повышении квалификации установленного образца (по ФЗ № 273) с использованием тестового курса

    В качестве примера бланка для сертификата в решении используется бланк действующей типографии (Киржачская типография).
    Если вы планируете настраивать сертификат самостоятельно, в уже существующем курсе — воспользуйтесь информацией в разделе «Настройка интеграции с ФИС ФРДО».

    Алгоритм решения:

    1) Импорт тестового курса из резервной копии.

    2) Проверка корректности выдачи удостоверения.

    Решение:

    Импорт курса «Техника безопасности и охрана труда (пример формирования удостоверения)» из резервной копии

    1) Скачать файл на компьютер.

    2) Перейти на страницу «Восстановление курса » (Администрирование -> Курсы -> Восстановление курса) и загрузить курс в настройке «Файл» скачанный файл резервной копии курса (поз. 1 Рис. 4.1.1). Нажать на кнопку «Восстановить» (поз. 2 Рис. 4.1.1).

    Рис. 4.1.1. Загрузка резервной копии курса.

    3) Нажать на кнопку «Продолжить» (поз. 1 Рис. 4.1.2) для того, чтобы подтвердить загрузку курса.

    Рис. 4.1.2. Подтверждение загрузки курса.

    4) Выбрать категорию курса = Разное (поз. 1 Рис. 4.1.3) и нажать на кнопку «Продолжить» (поз. 2 Рис. 4.1.3).

    Рис. 4.1.3. Выбор категории курса.

    5) Оставить настройки восстановления без изменений и нажать на кнопку «Далее» (поз. 1 Рис. 4.1.4).

    Рис. 4.1.4. Настройки восстановления курса.

    6) Оставить настройки курса без изменений и нажать на кнопку «Далее» (поз. 1 Рис. 4.1.5).

    Рис. 4.1.5. Настройки курса при импорте.

    7) Подтвердить все настройки импорта, нажав на кнопку «Приступить к восстановлению» (поз. 1 Рис. 4.1.6).

    Рис. 4.1.6. Подтверждение настроек восстановления.

    После этого курс будет восстановлен из резервной копии.

    Проверка корректности выдачи удостоверения

    1) Обратите внимание, после миграции СЭО 3KL на версию 4.1 такие настройки, как «Ширина текстового поля сертификата», «Минимальная высота текстового поля сертификата» и «Альтернативный серийный номер сертификата» не переносятся при восстановлении курса из бэкапа, поэтому их необходимо актуализировать вручную после импорта курса. Для этого нужно сначала включить режим редактирования в курсе (поз. 1 Рис. 4.1.7), затем перейти к редактированию настроек сертификата (поз. 2 Рис. 4.1.7).

    Рис. 4.1.7. Переход к редактированию настроек сертификата.

    На открывшейся странице уточнить следующие настройки (Рис. 4.1.8):

    Рис. 4.1.8. Настройки сертификата.

    Ширина текстового поля сертификата (см. поз. 1 Рис. 4.1.8) = 265
    Минимальная высота текстового поля сертификата (см. поз. 2 Рис. 4.1.8) = 154
    Альтернативный серийный номер сертификата (см. поз. 3 Рис. 4.1.8) = Порядковый номер с префиксом (настройка «Префикс» (см. поз. 4 Рис. 4.1.8) будет перенесена автоматически)

    В представленном варианте регистрационный номер сертификата будет формироваться из префикса (в примере - АА 77) и порядкового номера выданного сертификата (в примере - 0002), вы можете заменить префикс или оставить поле его настройки пустым, чтобы регистрационный номер состоял только из порядкового номера выданного сертификата. Подробнее об этих настройках читайте в статье «Сертификат СЭО 3KL» («Раздел «Другие настройки»).

    Сохранить внесенные изменения.

    2) Далее нужно зачислить на курс тестового слушателя, который получит сертификат. Для этого на главной странице курса нажать на кнопку «Участники» во вторичной навигации (поз. 1 Рис. 4.1.9), а затем — на кнопку «Зачисление пользователей на курс» (поз. 2 Рис. 4.1.9).

    Рис. 4.1.9. Страница зачисления пользователей на курс.

    В открывшемся модальном окне найти с помощью поиска нужного слушателя и выбрать его (поз. 1 Рис. 4.1.10), затем раскрыть все настройки зачисления, нажав на кнопку «Показать больше.../Показать меньше...» (поз. 2 Рис. 4.1.10), отметить опцию «Окончание обучения» (поз. 3 Рис. 4.1.10) и выбрать в ней дату окончания обучения (для примера задайте дату на один день позже зачисления на курс). После этого нажать на кнопку «Зачислить выбранных пользователей и глобальные группы» (поз. 4 Рис. 4.1.10).

    Рис. 4.1.10. Зачисление пользователя в курс.

    3) Авторизоваться в системе под аккаунтом тестового слушателя, войти в курс «Техника безопасности и охрана труда (пример формирования удостоверения)» и перейти к тесту «Тест для получения сертификата (удостоверения)» (поз. 1 Рис. 4.1.11).

    Рис. 4.1.11. Переход к Тесту.
    Рекомендуем входить непосредственно под тестовым пользователем, не используя фунцию «Зайти под именем», так как в этом случае сертификат может формироваться некорректно.

    4) Нажать на кнопку «Попытка теста» (Рис. 4.1.12), чтобы перейти непосредственно к тестированию.

    Рис. 4.1.12. Кнопка перехода к тестированию.

    На единственный вопрос Теста нужно ответить «Да» (Рис. 4.1.13), чтобы он считался завершенным успешно. Затем необходимо нажать на кнопку «Закончить попытку...». После этого у тестового пользователя появится возможность получить сертификат.

    Рис. 4.1.13. Ответ на вопрос Теста.

    5) Перейти к просмотру сертификата (поз. 1 Рис. 4.1.14).

    Рис. 4.1.14. Переход к просмотру сертификата.

    Нажать на кнопку «Получить сертификат» (поз. 1 Рис. 4.1.15), чтобы сформировать сертификат для тестового пользователя.

    Рис. 4.1.15. Кнопка получения сертификата.

    Тестовый пользователь получит сертификат следующего вида (Рис. 4.1.16).

    Рис. 4.1.16. Выданный тестовому пользователю сертификат.

    Рекомендации по изменению тестового курса под свои потребности:

    Если вы в дальнейшем планируете использовать этот курс в своей системе, вы можете:

    • Изменить название курса. Для того, чтобы это сделать, нужно со страницы курса нажать на кнопку «Настройки» (поз. 1 Рис. 4.1.17) во вторичной навигации и ввести новое название в поле настройки «Полное название курса» (поз. 2 Рис. 4.1.17). В примере это название = «Техника безопасности в работе среднего медицинского персонала».

    Рис. 4.1.17. Настройки курса.
    • Удалить демонстрационный Тест из курса. Для этого на главной странице курса перейти в режим редактирования (поз. 1 Рис. 4.1.18), нажать на три точки рядом с Тестом (поз. 2 Рис. 4.1.18) и выбрать опцию «Удалить» (поз. 3 Рис. 4.1.18).

    Рис. 4.1.18. Удаление демонстрационного Теста из курса.
    • Наполнить курс другими элементами курса, от которых будет зависеть выдача сертификата. В качестве примера добавим в курс элемент курса «Задание». Для этого на странице курса нужно перейти в режим редактирования (поз. 1 Рис. 4.1.19), нажать на кнопку «Добавить элемент или ресурс» (поз. 2 Рис. 4.1.19) и щелкнуть на иконку Задания (поз. 3 Рис. 4.1.19).

    Рис. 4.1.19. Добавление Задания в курс.

    Выполнить настройки Задания (Рис. 4.1.20):

    Рис. 4.1.20. Настройки Задания.

    Название задания (см. поз. 1 Рис. 4.1.20) = Очное тестирование
    Описание (см. поз. 2 Рис. 4.1.20) = Элемент создан для перенесения оценки за очное тестирование по технике безопасности
    Доступность (см. поз. 3 Рис. 4.1.20) = Все опции раздела не должны быть отмечены

    Нажать на кнопку «Сохранить и вернуться к курсу» (см. поз. 4 Рис. 4.1.20).

    • Внести правки в форму удостоверения. Для этого включить режим редактирования на главной странице курса (поз. 1 Рис. 4.1.21), нажать на три точки рядом с элементом курса «Сертификат СЭО 3KL» и выбрать опцию «Редактировать настройки» (поз. 2 Рис. 4.1.21).

    Рис. 4.1.21. Переход к настройкам сертификата.

    На открывшейся странице редактирования сертификата перейти к настройке «Текст сертификата» и заменить выделенные на Рис. 4.1.22 области своими значениями.

    Рис. 4.1.22. Редактирование текста сертификата.

    Вы можете также заменить формат сертификата, загрузив свое изображение подложки в настройке «Файл с изображением сертификата» (см. поз. 1 Рис. 4.1.22).

    В разделе настроек «Ограничение доступа» заменить имеющееся ограничение доступа по выполнению демонстрационного теста на ограничение доступа по выполнению элементов курса. В рассматриваемом примере необходимо выбрать в качестве элемента курса «Очное тестирование» (Рис. 4.1.23).

    Рис. 4.1.23. Настройка ограничения доступности.
