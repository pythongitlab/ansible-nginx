    1. Общая информация

    Отчет позволяет отобразить статистику обученности сотрудников организации по отделам и может быть востребован, в первую очередь, в дополнительном профессиональном и корпоративном обучении.

    Для корректного формирования отчета необходимо, чтобы у пользователей СЭО 3КL были:

    • заполнены поля профиля «Учреждение (организация)» и «Отдел» — это влияет на учет пользователей в отчете в целом;
    • определены даты окончания подписки на курсы (срок обучения пользователей на курсах ограничен), а в самих курсах настроены условия завершения курса — для достоверного вычисления значений столбцов «Специалистов завершивших подготовку» и «Специалистов не прошедших подготовку».

    Отчет доступен на уровне системы (при наличии соответствующих прав) и может быть выгружен в виде файла формата xls, ods или csv.

    Внешний вид отчета (наименования колонок и элементов интерфейса) может быть кастомизирован с учетом названий структурных подразделений, которые используются в вашей организации.

    2. Возможности

    Отчет «Сводка по обучению» (Администрирование -> Отчеты -> Сводка по обучению) отображает сводную информацию по прохождению курсов пользователями учебного портала (Рис. 2.1) и включает следующие данные:

    Рис. 2.1. Отчет «Сводка по обучению».
    • Отдел — список отделов (должностей) сотрудников организации. Значения берутся из поля профиля «Отдел».

    Отображаемый список отделов зависит от установленного фильтра по организациям (Рис. 2.3), который позволяет настроить отбор отделов по значению поля профиля «Учреждение (организация)» и выводить в отчет:

    • только отделы выбранной в фильтре организации (Рис. 2.2);

    Рис. 2.2. Отчет «Сводка по обучению» для выбранной организации «Итальянский ресторан».
    • все отделы всех организаций, которые проходят обучение на портале (Фильтр = Все) (см. Рис. 2.1).

    Рис. 2.3. Выпадающий список поля «Фильтр» с перечнем организаций, которые проводят обучение сотрудников на портале.
    Доступность фильтра (возможность просмотра отделов только своей организации или всех организаций) определяется наличием у пользователя определенного набора прав.
    • Количество специалистов — общее количество сотрудников данного отдела, зарегистрированных в СЭО 3КL. Пользователь будет отображаться и учитываться в общем количестве специалистов, если у него заполнены поля профиля «Отдел» и «Учреждение (организация)», и есть хотя бы одна подписка на курс.

    Ссылка «Количество специалистов» кликабельна и позволяет перейти к перечню специалистов выбранного отдела (Рис. 2.4).

    Рис. 2.4. Список специалистов отдела «Сомелье» во всех организациях.
    Возможность просмотра списка специалистов доступна при наличии права report/ot_usersoverview:view_detail.
    • Запланировано курсов обучения — суммарное количество подписок на курсы  пользователей данного отдела. Для вычисления запланированного количества курсов используются данные из Истории обучения обо всех имеющихся у пользователей активных подписках на курсы. При этом, если на момент формирования отчета уже есть слушатели, завершившие программу обучения (специалисты завершившие подготовку) и дата окончания подписки завершенных ими курсов меньше текущей даты, то курсы таких пользователей учитываться как запланированные не будут.
    Обратите внимание: если у пользователя есть курсы с неограниченной подпиской (без даты окончания), то такие курсы будут продолжать учитываться как запланированные, даже если слушатель полностью завершит программу обучения по всем курсам. При этом, сам пользователь будет учитываться в отчете как специалист завершивший подготовку.
    • Специалистов в процессе подготовки — количество пользователей, которые, в текущий момент, проходят программу обучения. Пользователь определяется как специалист в процессе подготовки, если у него:
      • есть хотя бы один курс с невыполненными условиями завершения и датой окончания подписки больше текущей даты;
      • нет ни одного просроченного курса (с завершенной подпиской, но не выполненными условиями завершения курса).
    • Специалистов завершивших подготовку — количество пользователей, которые завершили программу обучения (во всех запланированных курсах своевременно выполнены условия завершения курса).
    Если у пользователя есть завершенные курсы с неограниченной подпиской (без даты окончания), то такие курсы будут считаться «своевременно выполненными».
    • Специалистов не прошедших подготовку — количество пользователей, которые не выполнили программу обучения. Здесь учитываются пользователи, у которых есть хотя бы один курс в котором завершен период подписки (дата окончания подписки меньше текущей даты) и не выполнены условия завершения курса.

    Сформированный отчет можно сохранить на компьютере, выбрав необходимый формат файла (Рис. 2.4).

    3.1. Использование прав для фильтрации данных в отчете

    «Право просмотра всех фильтров» совместно с правами «Право просмотра по всем фильтрам» и «Право просмотра по личному фильтру» используются для организации иерархии просмотра:

    • пользователь не может просматривать отчет (вышеперечисленные права не выданы пользователю);
    • пользователь может просматривать отчет с отделами той организации, название которой совпадает со значением его поля профиля «Организация(учреждение)» (выдан набор прав report/ot_usersoverview:view + report/ot_usersoverview:view_my);
    • пользователь может просматривать отчет со всеми отделами всех организаций на учебном портале (набор прав report/ot_usersoverview:view + report/ot_usersoverview:view_all).

    Так сотруднику, ответственному за обучение непосредственно внутри дочерней организации компании, следует выдать «Право просмотра всех фильтров» + «Право просмотра по личному фильтру», которые позволят ему просматривать отчет по отделам только своей организации.

    А набор «Право просмотра всех фильтров» + «Право просмотра по всем фильтрам», выданный специалисту головного предприятия (ответственному за обучение всей компании), реализует возможность контроля обучаемости сотрудников по всем отделам всех организаций компании.

    4. Глобальные настройкитребуются права администратора

    Действия в данной инструкции с пометкой требуются права администратора могут быть выполнены только при наличии доступа «Полный Администратор». Начиная с 2022 года для вновь заключаемых договоров такой набор прав предоставляется роли «Диспетчер-администратор» по умолчанию. Если ваш договор был заключен раньше, вы можете обратиться в техподдержку с соответствующей заявкой на расширение набора прав диспетчера-администратора.
    Важно! Неосторожные действия полного администратора могут повредить систему и контент таким образом, что исправление последствий вмешательства выйдет за рамки гарантийной технической поддержки. Для безопасного изменения глобальных настроек мы рекомендуем выбрать набор требуемых опций по данной инструкции и обратиться в техническую поддержку с заявкой на конфигурирование системы.
    Подробная информация в статье «Полный административный доступ».

    4.1. Настройки плагина

    Глобальные настройки плагина доступны по следующему пути: Администрирование -> Плагины -> Отчеты -> Сводка по обучению (Рис. 4.1):

    Рис. 4.1. Включение периодического сбора данных для отчета.
    • Опция «Включить периодический сбор данных для отчета». Если не установлена, данные для отчета будут собираться в ручном режиме, по нажатию кнопки «Пересобрать данные» (см. Рис. ) в интерфейсе отчета.

    Если опция установлена, периодический сбор данных для отчета выполняется как регулярная задача report_ot_usersoverview\task\collectdata планировщика задач (Администрирование->Сервер->Задачи->Планировщик задач->Периодический сбор данных для отчета). В настройках задачи можно указать желаемый период автоматического обновления данных для отчета.

    Не рекомендуем включать и настраивать периодический сбор данных для отчета на объемных системах с большим количеством пользователей — выполнение задачи по пересчету достаточно ресурсоемкое и может приводить к значительному замедлению работы СЭО 3КL. Для получения актуальных данных используйте возможность ручного пересчета при помощи кнопки «Пересобрать данные» непосредственно в интерфейсе отчета (см. Рис. 2.1).

    4.2. Настройки СЭО 3КL (условия формирования отчета)

    Для правильного формирования отчета необходимо, чтобы:

    1) У пользователей СЭО 3КL были корректно заполнены поля профиля «Учреждение (организация)» и «Отдел» (Рис. 4.2.1).

    Рис. 4.2.1. Поля профиля «Учреждение (организация)» и «Отдел» на странице учетной записи пользователя.
    Дополнительная информация в статьях «Добавление кастомных (настраиваемых) полей в профиль пользователя», «Блок «Информация». Управление настраиваемыми полями в блоке «Информация» (MyInfo), поля профиля, личный кабинет».

    2) В курсах настроены условия завершения курса и срок обучения пользователей на курсах ограничен (установлена дата окончания подписки на курс).

    Подробная информация в статьях «Завершение курса. Настройки отслеживания выполнения для курса», «Подписка пользователей на курс. Дополнительная информация».

    5. Дополнительная информация

    5.1. Кастомизация отчета

    Названия столбцов таблицы и элементов интерфейса отчета могут быть изменены в соответствии с требованиями вашей организации.

    Для этого необходимо отредактировать следующие языковые строки языкового пакета ( Администрирование->Язык->Локальные изменения языкового пакета->Изменить языковой пакет) для компонента системы report_ot_usersoverview (Рис. 5.1.1):

    number_enrols_active Специалистов в процессе подготовки Строка в таблице отчета
    number_enrols_all Запланировано курсов обучения Строка в таблице отчета
    number_enrols_completed Специалистов завершивших подготовку Строка в таблице отчета
    number_enrols_failed Специалистов не прошедших подготовку Строка в таблице отчета
